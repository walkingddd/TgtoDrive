## ✨ 核心功能

### 🤖 115、123网盘自动分类整理

* **🎬 智能刮削与识别**：基于文件名通过 TMDB API 自动匹配元数据，告别无意义的乱码文件名。
* **🗂️ 策略分流体系**：内置高优先级分类逻辑，自动将资源归类为：
  * `综艺` (Variety)
  * `纪录片` (Documentary)
  * `动漫` (Anime)
  * `剧集` (TV Shows)
  * `电影` (Movies)
* 🌍 地区二级归类 (Secondary Classification)：
  * 深度整理：支持在电影、剧集等五大分类下，进一步按 国产、欧美、日韩、其他 创建子目录进行归类。
  * 灵活开关：一键控制启用或不启用，满足精细化整理需求。
* **♻️ 智能洗版 (Smart Dedupe)**：
  * 自动检测已存在的同名文件。
  * **按大小洗版**：可配置保留最大（高画质优先）或最小（省空间优先）版本，自动清理低质量重复文件。
* **🖥️ Web 可视化配置**：拒绝繁琐的配置文件修改，通过 Web 界面轻松管理 API Key、监控目录和整理规则。
* **🐳 Docker 部署**：开箱即用，支持 NAS 及各类服务器环境。

### 🎞️ STRM 媒体库能力

* **115、123 STRM 批量生成**：支持按目录批量生成 `.strm` 文件，并按网盘目录结构映射到本地媒体库。
* **整理后自动增量同步**：开启 `ENV_123_ORGANIZE_SYNC_STRM=1` 后，每轮 123 整理完成仅同步本轮真实发生变更的目录。
* **目录级去重 + 脏标记重试**：同一目录不会重复并发同步；若同步过程中再次发生变化，会自动重新入队。
* **元数据增量下载**：支持同步字幕、封面、NFO、音频等元数据文件，便于 Emby / Plex / 本地播放器识别。
* **失效 STRM 与空目录清理**：自动清理无效 `.strm` 文件和空目录；遇到扫描异常时会自动跳过清理，避免误删。
* **302 播放跳转**：支持通过 `/play/{file_id}/{etag}/{size}/{filename}` 动态解析真实地址，不在 STRM 中写死短效直链。
* **批量汇总通知**：整理结束后自动推送 TG 汇总，包含成功/异常目录、STRM 更新数、元数据下载数、清理统计，并支持树状明细分批发送。

### 🎬 Emby 反向代理302播放能力

* **Emby 反代面板**：在 Web 管理页新增 `系统工具 -> Emby反代` 页面，支持多 Emby 服务器卡片式管理。
* **单卡保存 / 单卡启停**：每个 Emby 反代实例支持独立“保存并应用”、“开启 / 关闭”、“测试”、“删除”，新增实例默认不会自动开启。
* **PlaybackInfo 劫持改写**：自动拦截 `/Items/{id}/PlaybackInfo`，将 `MediaSources[].DirectStreamUrl` 改写为网盘真实直链。
* **严格直链播放策略**：优先按 `/play/{file_id}` 解析 123 `download_info` 直链，失败后再回退 `get_download_url_by_path`；若仍失败则直接返回失败，不再回落本地代理播放。
* **多实例持久化配置**：反代配置持久化保存到 `db/embydata/emby_proxy_config.json`，重启后按各实例保存状态恢复。

### 🤖 自动化与监控

* **基础支持**：转存 123 / 115 / 天翼云盘链接，解锁并转存影巢链接。
* **全能频道监控**：实时监控 Telegram 频道，支持 **123、115、天翼云盘、影巢** 频道资源的自动捕获。
* **增量更新监控**：支持对指定的 **123网盘分享链接** 进行监控，自动转存新增的文件。
* **影巢/HDHive 深度集成**：
  * 支持直接转发 **影巢 (HDHive)** 频道消息进行转存。
  * 支持监控 **影巢 115 频道**。

### 📥 视频下载与离线

* **全网视频下载**：
  * **Bilibili**：直接发送 B站 链接给机器人，自动下载视频。
  * **抖音 (Douyin)**：直接发送 抖音 链接给机器人，自动下载无水印视频。
* **多协议离线下载**：
  * **磁力链 (Magnet)**：发送给机器人，自动提交至 123 或 115 离线列表。
  * **电驴 (ed2k)**：发送给机器人，自动提交至 115 离线列表。
  * **Torrent 文件**：直接发送 `.torrent` 种子文件给机器人，自动提交至 123 或 115 离线。

### 🛠️ 交互与转存工具

* **通过TG API实现万能一键转存**：
  * **万能转发**：无视私有频道禁止转发限制，直接以用户身份复制消息并发送至机器人，即可触发处理。
  * **跨盘黑科技**：
    * **115 -> 123**：使用指令 `/115to123`，将 115 分享秒传至 123，十万级文件量扫描只需十秒。
    * **115 CID -> 123**：使用指令 `/115cidto123`，指定 115 的文件夹秒传至 123，十万级文件量扫描只需十秒。
    * **天翼 -> 123**：使用指令 `/189to123`，将天翼资源秒传至 123。
    * **夸克 -> 123**：转发夸克链接，自动生成秒传转存至 123。
* **秒传神器**：支持转发 `JSON` 文件或发送秒传链接，支持 **PT本地文件** 扫描并无限尝试秒传至 123/115（防运营商制裁）。

### 🔍 搜索与社区

* **多源搜索**：
  * `/share 关键词`：搜索 123网盘内容并生成分享链接（支持一键发帖到123资源社区）。
  * `/pansou 关键词`：搜索 **pansou全网** 资源。

  * `/hdhive 关键词`：搜索 **影巢 (HDHive)** 板块资源，支持模糊搜索。

### ⚡ 媒体服务与运维

* **直链播放**：内置 Web Server，访问 `http://IP:12366/d/path` 获取 123 文件直链。
* **Emby 助手**：支持自动刷新 Emby 影视服务器中缺失的海报图片。
* **弹幕自动挂载**：支持 `misaka_danmu_server`，触发 302 播放时自动下载对应集及下一集的弹幕。
* **Web SSH 终端**：内置网页版远程终端，方便直接管理 NAS 或服务器文件。
* **天翼扫码登录**：支持通过 `/189login` 指令获取二维码，解决账密登录失败的问题。

## 🐳 Docker 快速部署 (推荐)

### 1. 准备工作

* 安装 Docker 20.10+ & Docker Compose 2.0+。
* **网络环境**：容器需要能访问 Telegram API（建议配置 HTTP 代理）。

### 2. 创建配置文件

在项目目录下创建 `docker-compose.yml`：

```yaml
version: '3'

services:
  tgto123-service:
    image: walkingd/tgto123:latest
    container_name: tgto123
    network_mode: host  # 推荐 host 模式以简化端口映射和直链访问
    environment:
      # --- 基础配置 ---
      - TZ=Asia/Shanghai
      # 必填：WEB管理页面的登录账号密码
      - ENV_WEB_PASSPORT=admin
      - ENV_WEB_PASSWORD=password

      # --- Telegram配置 ---
      # 必填：从 @BotFather 获取
      - ENV_TG_BOT_TOKEN=your_bot_token
      # 必填：从 @userinfobot 获取管理员ID
      - ENV_TG_ADMIN_USER_ID=123456789

      # --- 网络代理配置 (可选) ---
      # 若本机已开启全局透明代理，则无需配置以下三项
      # Clash用户通常填 http://127.0.0.1:7890
      - HTTP_PROXY=http://127.0.0.1:7890
      - HTTPS_PROXY=http://127.0.0.1:7890
      - NO_PROXY=localhost,127.0.0.1,192.168.0.0/16,10.0.0.0/8,123pan.com,115.com,189.cn,quark.cn

    volumes:
      # 数据库与日志持久化
      - ./db:/app/db
      # [推荐] B站、抖音等视频下载保存路径
      - ./downloads:/app/downloads
      # [可选] PT下载目录映射：左侧填NAS本地路径，右侧固定为 /app/upload
      # 用于实现本地文件秒传到网盘，不需要可去掉
      - /vol3/1000/Video/MoviePilot/transfer:/app/upload
      # [可选] STRM输出目录：用于保存 /app/strm 下生成内容,/vol1/1000/Emby/strm 改成你的目录
      - /vol1/1000/Emby/strm:/app/strm

    restart: always
```

### 3. 启动服务

```bash
docker-compose pull  # 拉取最新镜像
docker-compose up -d # 后台启动
```

### 4. ⚠️ 关键步骤：Web端初始化配置

容器启动后，**必须** 访问 Web 管理页面进行详细配置，否则大部分功能无法使用。

* **访问地址**: `http://你的NAS_IP:12366` (例如 `http://192.168.1.5:12366`)
* **配置内容**:
  * 填写网盘 Cookie/Token。
  * 配置监控频道（支持 123/115/天翼/影巢 等）。
  * 配置增量监控的分享链接。
  * 新增功能配置：WebSSH、Emby海报刷新、Emby反代、STRM同步等设置。

### 🎯 关键词白名单可视化配置（ENV_FILTER 系列）

Web 管理页已支持对以下变量进行“弹窗式”可视化配置：

* `ENV_FILTER`
* `ENV_FILTER_115`
* `ENV_FILTER_189`

使用方式：

1. 进入对应配置项，点击右侧 **「可视化配置」**。
2. 在弹窗内通过 **TMDB 搜索卡片** 添加规则，或使用「手动添加」补充关键词。
3. 每条规则改为更易理解的 3 个字段：
   * **影视名称**：支持输入一个或多个名称（可用 `|` 分隔同义词）。
   * **分辨率（下拉）**：`全部 / 4k / 1080p / 720p`。
   * **特效（下拉）**：`全部 / 杜比视界 / HDR / SDR`。
4. 系统会按下拉选项自动展开关键词（例如杜比视界会自动匹配 `Dolby Vision / DolbyVision / DV / DoVi`）。
5. 支持 **自定义正则卡片**（单输入框），用于直接补充复杂规则。
6. 系统会实时预览最终写回的正则表达式。
7. 点击「保存并回写」后，值会同步回对应的环境变量输入框。

兼容策略：

* **空规则 = 全部都存**：保存为空字符串，表示该场景不过滤。
* **高级正则模式**：可直接输入完整正则，原样保存。
* **兼容旧逻辑**：最终仍写回 `ENV_FILTER*` 字符串，不改变现有转存匹配流程。

### 🎬 Emby 反代配置页

最新版本已内置 Emby 反代页面，适合需要让 Emby Web / TV 客户端直接走网盘直链播放的场景。

#### 入口

登录 Web 管理后台后，进入：

* **系统工具** → **Emby反代**

#### 支持能力

* 支持添加 **多个 Emby 服务器**，每个服务器独立监听端口。
* 页面采用 **卡片式管理**，每张卡片对应一个 Emby 反代实例。
* 每张卡片都可以独立：
  * **保存并应用**
  * **开启 / 关闭**
  * **连接测试**
  * **删除**
* **默认不会自动开启**：新增卡片后，需要先点击“保存并应用”，再点击“开启”。

#### 工作原理

播放器访问反代端口时，程序会把普通请求转发到目标 Emby；当命中：

* `/Items/{id}/PlaybackInfo`

时，会自动重写 Emby 返回的播放信息：

* 把 `MediaSources[].DirectStreamUrl` 替换成 123 网盘真实直链
* 若部分源无法解析，会移除失败源，避免客户端回落到原链路
* 若全部解析失败，会直接返回失败，不走本地代理兜底

#### 直链解析优先级

当前直链获取策略如下：

1. 优先按 `/play/{file_id}` 调用 123 OpenAPI `download_info`
2. 若失败，则回退 `get_download_url_by_path`
3. 若仍失败，则直接返回失败

#### 配置保存位置

Emby 反代配置保存在：

* `db/embydata/emby_proxy_config.json`

其中会记录：

* Emby 地址
* API Key
* 监听端口
* 当前是否开启（`enabled`）

因此程序重启后，会按上次保存的状态恢复各个反代实例。

### 🎞️ STRM 使用说明

1. 在 Docker Compose 中挂载 STRM 目录：`- ./strm:/app/strm`。
2. 在 Web 配置页或 `.env` 中配置以下项目：
   * `ENV_123_STRM_FOLDER_ID`
   * `ENV_123_STRM_PLAY_BASE_URL`
   * `ENV_123_STRM_EXTS`
   * `ENV_123_STRM_DL_EXTS`
   * `ENV_123_STRM_INTERVAL`
3. 若希望每轮整理后自动同步 STRM，可开启：`ENV_123_ORGANIZE_SYNC_STRM=1`。
4. 115 分享可用中转接口：`/api/115strm?share_code=xxx&id=123&receive_code=yyy`。

#### 🔥 最新 STRM 特性（已实装）

* **整理后后台增量同步**：123 整理结束后自动等待后台 STRM 队列收尾，再统一输出结果。
* **目录级去重 + 脏标记重试**：同目录不会重复并发同步；若同步过程中目录再次发生变化，会自动重入队列。
* **垃圾清理与容错**：支持自动清理失效 `.strm` 与空目录；当扫描异常时自动跳过清理，优先保证数据安全。
* **结构化结果明细**：同步结果包含 STRM 更新、元数据下载、清理文件/目录、失败项等统计。
* **批量通知防刷屏**：明细默认按树状结构分批发送；超大批次会自动跳过明细，只发送汇总。
* **播放地址运行时生效**：`ENV_123_STRM_PLAY_BASE_URL` 变更后可在后续任务运行中生效，无需额外改代码。
